Backend API Requirements & Payload Specifications

Kalisia Sales CRM Integration

This document outlines the API endpoints, request/response payloads, and schema definitions required to connect the Kalisia Sales CRM frontend with the backend service. It is designed to match the current Zustand state management (crmStore.ts) and pages (/dashboard, /enquiries, /site-visits, /quotations, /lost-enquiries).

1. Global API Standards

  • Base URL: /api/v1 (or as per backend environment configuration)
  • Content Type: application/json for all requests and responses.
  • Naming Conventions: camelCase for JSON payload properties (matching the frontend TS interfaces).
  • Date Format: YYYY-MM-DD strings (e.g., "2026-06-29").
  • Time Format: HH:MM strings (e.g., "14:30").
  • Authentication: All endpoints should expect a bearer token (Authorization: Bearer <token>) in headers unless specified otherwise.
  • Error Codes:
  • 400 Bad Request: Validation failure.
  • 401 Unauthorized: Authentication failed.
  • 403 Forbidden: Insufficient permissions.
  • 404 Not Found: Resource not found.
  • 500 Internal Server Error: Server failure.

  • HTTP Method: GET
  • Route: /dashboard/counters
  • Response Payload (200 OK):
JSON Payload / Schema 
{
  "newEnquiries": 3,

  "validUntil": "2026-07-23",
  "enquiryNo": "ENQ-001",

6.3 Update Quotation (Draft/Sent)

  • HTTP Method: PUT or PATCH
  • Route: /quotations/:quotationNo
  • Request Payload:
  • Allows editing pricing, validity, terms, active theme, and remarks.
  • Can also accept a status change (e.g. status upgraded to "Sent" or "Accepted").
  • Response Payload (200 OK):
  • Returns the updated Quotation.

6.4 Create Quotation Revision

When negotiating prices, the backend must save the current state as a revision and increment the quotation version. - HTTP Method: POST - Route: /quotations/:quotationNo/revisions - Request Payload:

JSON Payload / Schema 
{
  "revisionReason": "Negotiated discount and scope adjustment",
  "remarks": "Active revision version.",
  "materialCost": 75000,
  "labourCost": 25000,
  "transportation": 5000,
  "miscellaneous": 3000,
  "discount": 5000,
  "tax": 5150,
  "grandTotal": 108150,
  "pricingMethod": "manual",
  "attachedTemplate": "",
  "selectedThemeId": "theme-1"
}
  • Backend Action:
  • Retrieve current quotation.
  • Set the current details of the quotation as a new entry in its revisions list with version number matching the previous currentVersion.
  • Increment currentVersion by 1.
  • Overwrite the main Quotation's pricing block and active fields with the parameters supplied in the request.
  • Response Payload (200 OK):
  • Returns the updated parent Quotation object containing the new revision history and incremented version.

6.5 Activate Specific Revision Version

  • HTTP Method: POST
  • Route: /quotations/:quotationNo/revisions/:version/activate
  • Request Payload: (None required)
  • Backend Action:
  • Restores the primary pricing and configuration fields of the quotation to match the data stored in the specified version entry of the revisions array.
  • Response Payload (200 OK):
  • Returns the updated parent Quotation object.

6.6 Delete Revision Version

  • HTTP Method: DELETE
  • Route: /quotations/:quotationNo/revisions/:version
  • Response Payload (200 OK):
  • Returns updated parent Quotation object.

6.7 Delete Quotation

  • HTTP Method: DELETE
  • Route: /quotations/:quotationNo
  • Response Payload (200 OK):
JSON Payload / Schema 
{ "message": "Quotation deleted." }

7. Lost Opportunities API

Tracks closed-lost prospects and supports reopening them.

7.1 List Lost Enquiries

  • HTTP Method: GET
  • Route: /lost-enquiries
  • Query Parameters:
  • search (string, optional) - Filters by company name, enquiry number, or reason.
  • lostStage (string, optional) - Stage when lost. Enum: ALL, Enquiry, Site Visit, Quotation.
  • lostReason (string, optional) - Enum: ALL, Price Too High, Competitor Won, Project Cancelled, Budget Issue, No Response, Duplicate, Client Not Interested, Scope Changed, Other.
  • Response Payload (200 OK):
JSON Payload / Schema 
[
  {
    "lostNo": "LST-001",
    "enquiryNo": "ENQ-004",
    "companyName": "Capital Tower Offices", // Note: frontend uses lost.companyName & lost.company. Backend should standardize.
    "contactPerson": "Robert Chen",
    "project": "Level 14 Fit-out",
    "lostStage": "Quotation",
    "lostDate": "2026-06-25",
    "lostReason": "Price Too High",
    "remarks": "Client went with local contractor offering 25% lower price.",
    "followUpDate": "2026-12-01",
    "canReopen": true,
    "originalStatus": "Quotation"
  }
]

7.2 Reopen Lost Opportunity

  • HTTP Method: POST
  • Route: /lost-enquiries/:enquiryNo/reopen
  • Request Payload: (None required)
  • Backend Action:
  • Locates the related Enquiry.
  • Changes Enquiry status back to its originalStatus (e.g. "Quotation" or "Enquiry").
  • Deletes the LostEnquiry log record.
  • Response Payload (200 OK):
JSON Payload / Schema 
{
  "message": "Opportunity reopened successfully.",
  "enquiryNo": "ENQ-004",
  "restoredStatus": "Quotation"
}

7.3 Delete Lost Log

  • HTTP Method: DELETE
  • Route: /lost-enquiries/:lostNo
  • Response Payload (200 OK):
JSON Payload / Schema 
{ "message": "Lost opportunity retention log deleted." }

  • Route: /settings/themes

  • HTTP Method: PUT
  • Route: /settings/themes/:id
  • Request Payload:

8.4 Delete Custom Theme

  • HTTP Method: DELETE
  • Route: /settings/themes/:id
  • Response Payload (200 OK):
  • Deletes the theme. System themes must block deletion.
